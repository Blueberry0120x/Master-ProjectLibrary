## 6. Key Procedures

### 6a. Adding a New Site (GIS Intake)
1. Search address in KC Assessor (WA) or ParcelQuest (CA) ‚Üí get APN and lot SF
2. Get lat/lng from Google Maps satellite: right-click **parcel centroid** (not street door)
3. Extract parcel polygon from GIS portal (KC iMap / SanDAG) ‚Äî 4+ vertices in lat/lng
4. Compute true centroid from polygon (shoelace formula); use as `saved.lat/lng`
5. Write `data/sites/{state}-{id}_{name}.json` with site + saved + checklist keys
6. Register in `data/sites/index.json`
7. Set `activeSiteId` in `data/site-data.json`
8. Run `py tools/build.py`
9. Verify in browser ‚Äî parcel center must land on the actual parcel (not road)

**Critical lessons from this session:**
- Geocoders return the street address point, not parcel centroid. For a 132 ft deep lot, that's the front door ‚Äî the center is ~66 ft back.
- KC GIS spatial bbox query (`geometryType=esriGeometryEnvelope`) reliably returns parcel polygon and centroid when address search fails.
- KC APN prefix (MAJOR) does NOT always match the township/range prefix ‚Äî Renton lot 7229900345 has MAJOR 722990, not 132305 as assumed from neighbor data.

### 6b. Fixing a Wrong Parcel Location
1. Do NOT trust geocoded coordinates for `saved.lat/lng`
2. Run KC GIS bbox query around the approximate area ‚Üí get polygon rings
3. Compute shoelace centroid from polygon vertices
4. Update `saved.lat`, `saved.lng`, `saved.parcelPolygon`
5. Update `site.lotWidth`, `site.lotDepth` from polygon measurements
6. Rebuild + verify

### 6c. Arc Dimension Algorithm (Both Engines)
- Constants: `ARC_DEGEN=0.1`, `ARC_MAX_SEG=6`, `ARC_MIN_RUN=4`, `ARC_LEN_RATIO=2.0`
- Two copies: `engine-map.js:updateDimLabels()` and `engine-setback.js:drawSetbacks()`
- **Must stay identical.** `_` prefix in setback engine is intentional (scope collision avoidance).
- Label format: `arcLen.toFixed(1) + ' FT \u2312'` (‚åí)
- 4-pass: raw segs ‚Üí group arc/straight ‚Üí wrap-around merge ‚Üí build output

### 6d. Save Architecture
- `ExportEngine._payload()` is the ONLY source of truth for persisted state
- Every field that survives sessions must appear in `_payload()`
- `POST /save` preserves `.site`, overwrites `.saved` only
- Save triggers: Save Setbacks, Save Config, Save Boundary, Save to File, FAB Save, Lock toggle, drag-end

### 6e. Engine Contract (Non-Negotiable)
| Engine | Owns |

| ConfigEngine | All mutable state storage/retrieval |
