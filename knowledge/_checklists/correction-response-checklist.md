# Correction Response Checklist

Use this checklist when preparing a response to a plan check correction letter.
A clean, complete response is the single best way to achieve 2-cycle approval.

---

## 1. Before You Start

- [ ] Read every correction comment — do not skim
- [ ] Sort corrections by discipline (Building, Zoning, Fire, Stormwater, etc.)
- [ ] Identify comments that require design changes vs. notes-only vs. clarifications
- [ ] Identify comments that reference specific code sections — look them up, don't guess
- [ ] Flag any comments that are unclear — schedule a pre-response call with the plan checker

---

## 2. Response Narrative Document

- [ ] Response document includes: project address, PRJ/permit number, cycle number, date
- [ ] Each comment listed in order with its original comment number / discipline tag
- [ ] For each comment: write a direct, specific response — "See Sheet A-1, Note 3" is not enough
- [ ] State what you changed and why it satisfies the code requirement cited
- [ ] If you disagree with a comment: cite the specific code section that supports your position;
  request a meeting with the plan checker before submitting a formal disagreement
- [ ] Do not write "acknowledged" as a response — every comment needs a resolution statement

---

## 3. Revised Plans

- [ ] Revision clouds (bubbles) on all changed areas — every correction must be clouded
- [ ] Delta triangle with cycle number (Δ1, Δ2) in or near each cloud
- [ ] All sheets with revisions are re-issued with current date
- [ ] Sheet index updated to reflect reissued sheets
- [ ] No revision clouds from prior cycles removed — previous clouds remain until final approval
- [ ] Title block on all sheets shows revision date and cycle number

### Common Missed Locations

- Cover / summary sheet (FAR, setbacks, parking summary updated)
- Site plan (if setback, BMP, or utility change)
- Foundation plan (if soils or structural change)
- Structural details (if shear/hold-down change)
- CF1R / energy callouts on floor plans
- CALGreen checklist (if measure updated)

---

## 4. Code-Specific Response Tips

### Zoning Comments

- Cite the specific SDMC section (or equivalent) and show the calculation.
- Example: "Per SDMC 131.0446, FAR for RS-1-7 on a 5,000 sf lot is limited to 0.60 × 5,000 = 3,000 sf. Proposed primary dwelling = 1,800 sf, ADU = 1,200 sf (excluded from FAR per state ADU law Gov Code 65852.2). Total counted GFA = 1,800 sf = 0.36 FAR — compliant."

### Structural Comments

- Attach updated structural calculations if required.
- Show all revised hold-down schedules and shear panel tables.
- Resubmit full calculation packet if calculator changes.

### Fire Comments

- Provide manufacturer cut sheets for rated assemblies if requested.
- Show all smoke/CO detector locations on updated floor plan.
- Note deferred submittal items explicitly.

### Energy / Title 24 Comments

- Re-run and reprint CF1R if any change was made to envelope, mechanical, or water heating.
- Coordinate CF1R with plans — every discrepancy between CF1R and plan notes is a correction.

### CALGreen Comments

- Update checklist to match current plans.
- Add code section notes to plans where required (e.g., "VOC compliant per CALGreen §4.504.1").

### Stormwater Comments

- Update BMP site plan if site changes.
- Provide calculations if WQMP or hydromodification is questioned.

---

## 5. Submittal Package Assembly

### Required Items

- [ ] Written response narrative (PDF)
- [ ] Revised plan set (single bookmarked PDF — one file, all sheets)
- [ ] Any supplemental calculations (structural, energy, drainage)
- [ ] Any new forms required by the correction letter

### For San Diego (DSD/Accela)

- [ ] Upload sequence: RESPONSE → PLANS → SUPPLEMENTS
- [ ] File names must meet Accela requirements (no special characters, < 100 chars)
- [ ] All sheets in one bookmarked PDF; bookmarks named by sheet number and title
- [ ] Confirm upload was accepted in Accela before close of business

### For Other Jurisdictions

- [ ] Verify current resubmittal procedure with Building Division (some require counter drop-off)
- [ ] Confirm fee for second/subsequent cycles (if applicable)

---

## 6. After Submission

- [ ] Record submission date and cycle number in project log
- [ ] Confirm review time estimate with Building Division
- [ ] Check portal for status updates; don't wait for a call — check yourself
- [ ] If a cycle is taking longer than the published timeline, call the building department
  and ask for a status check (politely — plan checkers have heavy workloads)
- [ ] When corrections are issued: repeat this checklist from step 1

---

## Common Mistakes That Create Extra Cycles

| Mistake | How to Avoid |
|---------|-------------|
| Clouding changes but not updating plans to match | Cross-check every cloud against the response narrative |
| Re-running CF1R but not updating plan callouts | CF1R and plan notes must be identical |
| Addressing 9 of 10 comments and missing one | Read the letter three times before submitting |
| Writing vague responses ("see revised plans") | Every response must state what code requirement is met and how |
| Not removing prior-cycle clouds from PDF | Keep all clouds through final approval |
| Wrong file upload order in Accela | RESPONSE first, then PLANS |
